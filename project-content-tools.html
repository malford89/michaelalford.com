Broken Tractor LLC · 2025

AI Listing Generator

Role: Full-Stack Developer Stack: React, Node.js, Anthropic Claude API Built solo in under 30 days

The Problem

Broken Tractor's catalog had grown to over 19,000 products and 3,000 categories — most with incomplete or missing descriptions, titles, and metadata. Writing and publishing listings manually would have taken months of full-time work. The business needed a scalable solution that could produce consistent, on-brand copy at volume without sacrificing quality.

The Solution

I designed and built a full-stack internal web application — the Listing Generator — that uses the Anthropic Claude API to automatically generate product titles, descriptions, and category copy at scale. The app pulls product data, sends it to the Claude API with carefully engineered prompts, and returns polished, publish-ready content.

How It Works

The front end is a React application that lets internal users select products or categories, preview AI-generated content, make edits, and approve listings for publishing. The back end is a Node.js + Express API that handles authentication, communicates with the Anthropic Claude API, and syncs approved content back to BigCommerce via the REST API.

Prompt engineering was a core part of the work — crafting system prompts that consistently produced the right tone, format, and keyword structure for tractor parts content at scale.

Stack & Tools

React Node.js Express Anthropic Claude API BigCommerce REST API Prompt Engineering Claude Code

Outcome

Delivered solo in under 30 days. The app processed over 19,000 products and 3,000 categories — work that would have taken months manually. It became a core internal tool for ongoing catalog maintenance and new product onboarding at Broken Tractor.
